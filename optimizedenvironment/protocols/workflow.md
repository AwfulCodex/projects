# 工作流规则 (Agent Orchestration Workflow)

> 协议版本: 1.0.0
> 维护者: 编排器
> 参考: `protocols/internal-comms.md`, `protocols/message-format.md`

---

## 1. 标准工作流 (Standard Workflow)

```
用户输入
   │ (English)
   ▼
编排器 (Orchestrator)
   │ 加载: ORCHESTRATOR.md → shared/constraints.md → protocols/
   │
   ▼
逻辑代理 (logic-agent)
   │ 分析请求，生成任务计划
   │ 语言: 中文
   ▼
实施代理 (implementation-agent)
   │ 执行任务计划，记录结果
   │ 语言: 中文
   ▼
翻译代理 (translator-agent)
   │ 转换结果: 中文 → 英文
   ▼
用户界面代理 (user-facing-agent)
   │ 格式化并呈现给用户
   │ 语言: 英文
   ▼
用户输出
```

---

## 2. 错误恢复工作流 (Error Recovery Workflow)

```
实施代理 检测到错误
   │
   ▼
发送 【错误报告】 给逻辑代理
   │
   ▼
逻辑代理 分析错误
   │
   ├── 可修复 ──► 发送修订任务计划给实施代理
   │                    │
   │                    ▼
   │               实施代理 重试
   │
   └── 不可修复 ──► 发送 【升级请求】 给编排器
                        │
                        ▼
                   编排器 通知用户界面代理
                   (路由至翻译代理→用户界面代理)
```

---

## 3. 澄清工作流 (Clarification Workflow)

```
逻辑代理 检测到歧义
   │
   ▼
发送 【澄清查询】 给编排器
   │
   ▼
编排器 路由给翻译代理
   │
   ▼
翻译代理 将查询翻译为英文
   │
   ▼
用户界面代理 向用户提问
   │
   ▼
用户提供回答 (英文)
   │
   ▼
编排器 路由回逻辑代理
   │
   ▼
逻辑代理 继续规划（带有澄清信息）
```

---

## 4. 破坏性操作工作流 (Destructive Operation Workflow)

```
实施代理 识别破坏性步骤
   │ (删除, 覆盖, 强制推送, 重置)
   ▼
暂停执行
   │
   ▼
发送 【确认请求】 给编排器
   │
   ▼
编排器 → 翻译代理 → 用户界面代理 → 用户
   │
   ├── 用户确认 "yes"
   │      │
   │      ▼
   │   编排器 → 实施代理: 继续执行
   │
   └── 用户拒绝 "no"
          │
          ▼
       编排器 → 逻辑代理: 标记步骤已跳过，重新规划
```

## 5. 任务状态机 (Task State Machine)

```
[初始] ──► [规划中] ──► [执行中] ──► [翻译中] ──► [完成]
                │             │
                ▼             ▼
           [等待澄清]    [错误恢复]
                │             │
                ▼             ├── [重规划] ──► [执行中]
           [继续规划]         │
                              └── [升级] ──► [人工干预]
```

状态转换规则:

| 当前状态  | 触发条件           | 下一状态  |
|-----------|--------------------|-----------|
| 初始      | 收到用户请求       | 规划中    |
| 规划中    | 计划完成           | 执行中    |
| 规划中    | 检测到歧义         | 等待澄清  |
| 等待澄清  | 收到用户回答       | 规划中    |
| 执行中    | 所有步骤完成       | 翻译中    |
| 执行中    | 步骤失败           | 错误恢复  |
| 执行中    | 需要确认           | 等待确认  |
| 等待确认  | 用户确认           | 执行中    |
| 等待确认  | 用户拒绝           | 规划中    |
| 错误恢复  | 可修复             | 规划中    |
| 错误恢复  | 3次失败            | 人工干预  |
| 翻译中    | 翻译完成           | 完成      |

## 6. 代理启动检查清单 (Agent Startup Checklist)

每个代理在处理任何消息之前必须完成以下检查:

```
□ 已读取自身转向文件 (agents/[id]/steering.md)
□ 已加载 shared/constraints.md
□ 已加载 protocols/internal-comms.md
□ 已加载 protocols/message-format.md
□ 已知晓自身在工作流中的位置
□ 已确认输出目标代理
□ 语言配置正确（中文内部 / 英文用户界面）
```

## 7. 并行执行规则 (Parallel Execution Rules)

逻辑代理可以在任务计划中标记可并行的步骤:

```
步骤3a. 步描: [操作A]
   并行组: P1
步骤3b. 步描: [操作B]
   并行组: P1
```

- 同一并行组的步骤由实施代理并行执行。
- 并行组内的步骤不得有相互依赖。
- 并行步骤完成后，结果合并为单一状态报告。

## 8. 超时与重试策略 (Timeout and Retry Policy)

| 操作类型   | 超时  | 最大重试次数 | 升级阈值 |
|------------|-------|--------------|----------|
| 文件读写   | 30秒  | 2            | 3次失败  |
| 命令执行   | 120秒 | 1            | 2次失败  |
| 网络请求   | 60秒  | 2            | 3次失败  |
| 代理间消息 | 30秒  | 1            | 2次超时  |
