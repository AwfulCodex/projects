# 内部通信协议 (Internal Communication Protocol)

> 协议版本: 1.0.0
> 适用范围: 所有内部代理间通信
> 语言: 中文（强制执行）

---

## 1. 目的 (Purpose)

本协议规定所有代理间内部通信必须使用中文（简体）。
这一设计决策减少了令牌使用量，并将内部系统语言与用户界面语言严格分离。

English note: All internal agent-to-agent messages MUST be written in Simplified Chinese.
Only the User-Facing Agent produces English output. Violation of this rule is a
protocol breach and the Orchestrator must flag it.

---

## 2. 强制语言规则 (Mandatory Language Rules)

| 通信方向                          | 语言 | 执行级别 |
|-----------------------------------|------|----------|
| 逻辑代理 → 实施代理               | 中文 | 强制     |
| 实施代理 → 翻译代理               | 中文 | 强制     |
| 实施代理 → 逻辑代理（错误）       | 中文 | 强制     |
| 翻译代理 → 用户界面代理           | 英文 | 强制     |
| 用户界面代理 → 用户               | 英文 | 强制     |
| 任何代理 → 编排器（系统消息）     | 中文 | 强制     |

---

## 3. 令牌优化策略 (Token Optimization Strategies)

### 3.1 字段标签缩写
内部消息使用缩写字段名以减少令牌消耗:

| 完整标签     | 缩写标签 | 含义              |
|--------------|----------|-------------------|
| 任务标识符   | 任ID     | Task ID           |
| 执行状态     | 状态     | Status            |
| 步骤描述     | 步描     | Step description  |
| 错误类型     | 错类     | Error type        |
| 预期输出     | 期出     | Expected output   |
| 约束条件     | 约束     | Constraints       |
| 完成标准     | 完标     | Completion criteria |
| 执行代理     | 执代     | Executing agent   |

### 3.2 状态代码
使用单字或双字状态代码:

| 代码 | 含义      |
|------|-----------|
| 完   | 完成      |
| 失   | 失败      |
| 部   | 部分完成  |
| 等   | 等待中    |
| 进   | 进行中    |
| 停   | 已停止    |

### 3.3 优先级代码

| 代码 | 含义 |
|------|------|
| 高   | 高优先级 |
| 中   | 中优先级 |
| 低   | 低优先级 |
| 紧   | 紧急     |

## 4. 消息头规范 (Message Header Specification)

每条内部消息必须以标准消息头开始:

```
【消息】
发: [发送代理ID]
收: [接收代理ID]
型: [任务|结果|错误|查询|确认|升级]
优: [高|中|低|紧]
时: [ISO8601时间戳]
版: [消息版本号]
```

字段说明:
- `发` — 发送者 (sender)
- `收` — 接收者 (receiver)
- `型` — 消息类型 (message type)
- `优` — 优先级 (priority)
- `时` — 时间戳 (timestamp)
- `版` — 版本 (version)

---

## 5. 消息类型定义 (Message Type Definitions)

### 5.1 任务消息 (`型: 任务`)
由逻辑代理发送给实施代理。
包含完整任务计划。见 `protocols/message-format.md`。

### 5.2 结果消息 (`型: 结果`)
由实施代理发送给翻译代理。
包含执行结果摘要。见 `protocols/message-format.md`。

### 5.3 错误消息 (`型: 错误`)
由任何代理发送给逻辑代理或编排器。
包含错误详情和建议操作。

### 5.4 查询消息 (`型: 查询`)
由逻辑代理发送，表示需要用户澄清。
路由: 逻辑代理 → 编排器 → 用户界面代理 → 用户。

### 5.5 确认消息 (`型: 确认`)
在破坏性操作前由实施代理发出。
必须等待确认回复才能继续。

### 5.6 升级消息 (`型: 升级`)
在多次失败后由任何代理发给编排器。
触发人工干预流程。

---

## 6. 中文写作标准 (Chinese Writing Standards)

- 使用简体中文（不使用繁体）。
- 技术词汇优先使用既有中文术语（见 `shared/glossary.md`）。
- 无对应中文术语时，保留英文原词。
- 句子简洁，避免冗余修饰语。
- 使用项目符号列表，避免长段落。
- 数字: 使用阿拉伯数字（1, 2, 3）而非汉字数字（一, 二, 三）。

---

## 7. 协议违规处理 (Protocol Violation Handling)

若检测到内部消息使用英文:

1. 编排器记录违规事件。
2. 拒绝处理违规消息。
3. 向发送代理返回 `【协议错误】` 通知。
4. 要求代理重新发送符合规范的中文消息。

```
【协议错误】
错误代码: LANG-001
描述: 内部消息必须使用中文
违规代理: [agent-id]
要求: 请用中文重新发送消息
```
